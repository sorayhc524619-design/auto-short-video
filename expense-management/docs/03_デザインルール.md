# デザインルール v0.1

**対象ファイル**:`副業経費管理キット_v1.0.xlsm`
**前提ドキュメント**:`01_シート構成.md` / `02_列定義.md`
**作成日**:2026-04-20

このドキュメントでは、ファイル全体で統一する「色・フォント・レイアウト」の基本ルールを決めます。一度これを決めておくと、どのシートも同じ見た目になり、購入者が迷わず使えます。

---

## 1. カラーパレット(全体で8色のみ使用)

Excelで色を多く使いすぎると「おもちゃっぽく」なります。商品らしい落ち着きを出すため、**役割ごとに色を限定** します。

| 用途 | 色名 | RGB | 16進 | Excelのカラーピッカー位置 |
|---|---|---|---|---|
| **メインカラー** | 濃紺(ネイビー) | 31, 78, 121 | `#1F4E79` | 「標準の色」上から2段目の青系 |
| **アクセント** | 水色 | 197, 224, 248 | `#C5E0F8` | 「標準の色」薄い青 |
| **成功(🟢)** | 緑 | 112, 173, 71 | `#70AD47` | 「標準の色」緑 |
| **注意(🟡)** | 黄橙 | 237, 176, 33 | `#EDB021` | 「標準の色」黄 |
| **警告(🔴)** | 赤 | 192, 0, 0 | `#C00000` | 「標準の色」濃い赤 |
| **免責バナー背景** | 薄赤 | 248, 215, 218 | `#F8D7DA` | 自分でRGB指定 |
| **入力可能セル背景** | 薄黄 | 255, 248, 220 | `#FFF8DC` | 自分でRGB指定 |
| **ヘッダー背景** | 濃紺(メインと同じ) | 31, 78, 121 | `#1F4E79` | - |
| **ヘッダー文字** | 白 | 255, 255, 255 | `#FFFFFF` | - |
| **通常の文字** | 黒に近いグレー | 64, 64, 64 | `#404040` | - |

**使い分けルール**
- 購入者が値を入力できるセル → **薄黄(`#FFF8DC`)** 背景
- 数式・計算結果セル → 背景なし(白)
- 表のヘッダー → **濃紺背景・白文字・太字**
- グラフの主系列 → **濃紺**、副系列 → 水色
- 免責事項 → 薄赤背景+濃い赤文字で目立たせる

## 3. フォント

| 用途 | フォント | サイズ | 太字 |
|---|---|---|---|
| シートタイトル(各シートのB2セル) | メイリオ | 16pt | ○ |
| 表のヘッダー行 | メイリオ | 11pt | ○ |
| 通常のデータセル | メイリオ | 10pt | × |
| 大数字表示(ダッシュボードのサマリー) | メイリオ | 28pt | ○ |
| 判定結果の絵文字+テキスト | Segoe UI Emoji + メイリオ | 20pt | ○ |
| 注意書き・免責事項 | メイリオ | 9pt | × |
| ボタン内テキスト | メイリオ | 11pt | ○ |

**フォント選定理由**
- メイリオはWindows標準搭載で、ほぼ全環境で同じ見た目になる
- 游ゴシックは綺麗だが太字が薄く見えるため採用しない
- MSゴシックは古く見えるため採用しない

## 4. 行・列のサイズ

| 要素 | サイズ |
|---|---|
| データ行の高さ | 20pt |
| ヘッダー行の高さ | 28pt |
| タイトル行の高さ | 36pt |
| 既定の列幅(特記なしの場合) | 12 |
| 各列の個別幅 | `02_列定義.md` 参照 |
| 1行目の左右に空白列A | 列幅 2(余白感を出すため) |

**余白のつけ方**
- すべてのシートで、A列は「細い余白列」(幅2)にする
- コンテンツは B列から配置
- これにより左端に詰まった印象がなくなる

## 7. ボタンのデザイン

シート内に設置するボタン(マクロ実行用)は、**画像ではなく「図形(角丸四角形)」** で作ります。理由:後で文字変更・色変更がしやすいため。

**統一仕様**
- 形:角丸四角形
- サイズ:幅 160 × 高さ 36 ピクセル
- 塗りつぶし:濃紺(`#1F4E79`)
- 文字色:白
- フォント:メイリオ 11pt 太字
- 枠線:なし
- ホバー時の変化:なし(Excelは標準でホバー変化できないため)

配置するボタン一覧(`00_スタート` シート上):

|---|---|
| レシート入力へ | `01_入力` |

| ダッシュボード | `04_ダッシュボード` |
| 科目別集計 | `05_科目別集計` |
| 20万円判定 | `06_20万円判定` |

```

│ ⚠ 本ツールの判定は参考情報です。住民税・医療費控除・給与2,000万円超 │
│   の場合など例外があります。最終判断は税務署または税理士にご確認を  │
│   お願いします。ツールの判定結果に対する責任は負いかねます。       │
└────────────────────────────────────────────────────────┘
```

- 背景:`#F8D7DA`(薄赤)
- 文字色:`#C00000`(濃い赤)
- フォント:メイリオ 10pt 太字
- 配置先:`00_スタート` 下部 / `06_20万円判定` 上部 / `99_使い方` 末尾

## 次のステップ

このデザインルールにOKが出たら、M1④ **xlsmファイルのひな形作成** に進みます。
そこからは実ファイル(.xlsm)を作る作業になりますが、**私(Claude)はLinux環境で動いているため、完全な .xlsm ファイルは作れません**(VBAマクロは Windows Excel 上でのみ埋め込み可能)。

対応方針を2つご提案します:

**A案:「空のひな形 .xlsx」を私が作成 → 大久保さんがExcelで開いて .xlsm 形式で保存**
- 私の役割:シート名・タブ色・ヘッダー行・列幅・基本書式までを .xlsx として自動生成
- 大久保さんの作業:ファイルを開き「名前を付けて保存」→形式を「Excelマクロ有効ブック(.xlsm)」に変更(1分で完了)
- メリット:ゼロから手作業で作らなくて済む

**B案:ステップバイステップの手順書を私が作成 → 大久保さんが一から手動でExcelを操作**
- 私の役割:「シート1を追加→名前を `00_スタート` に→タブ色を……」という詳細手順書を作る
- 大久保さんの作業:手順書に従って Excelを操作(2〜3時間)
- メリット:Excel操作に慣れる・全て自分で把握できる

**おすすめは A案** です。Pythonライブラリ(openpyxl)でひな形ファイルを生成し、大久保さんは.xlsmに保存するだけで済みます。

M1④ 着手前に、方針をご判断ください。
